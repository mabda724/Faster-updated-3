## 11. Accessibility (CRITICAL)

### Minimum Standards (WCAG AA)

| Criterion | Requirement |
|-----------|------------|
| Text Contrast | 4.5:1 (normal), 3:1 (large 18+ or bold 14+) |
| Touch Target | 44x44dp minimum |
| Focus Indicator | 2-4px visible ring |
| Animation | Respect `prefers-reduced-motion` |
| Screen Reader | All icons have `Semantics(label:)` |
| Color Independence | Never use color alone to convey info |
| Dynamic Text | Layout must not break at max text scale |

### Semantic Labels

All interactive elements MUST have `Semantics` or `Accessibility` labels:
- Icon buttons → `Semantics(label: "description", child: Icon(...))`
- Status indicators → `Semantics(label: "Order status: completed", child: statusWidget)`
- Rating stars → `Semantics(label: "Rate $x out of 5", child: starRow)`

## 13. Anti-Patterns (AVOID)

| Anti-Pattern | Instead |
|-------------|---------|
| Emoji as icons | Use Material Icons |
| Placeholder-only labels | Use visible labels above inputs |
| Color-only status indicators | Add icon + text + color |
| Raw hex in components | Use semantic color tokens |
| Random spacing values | Use spacing token system |
| Hover-only interactions (mobile) | Use tap/press feedback |
| Instant state changes (0ms) | Use 150-300ms transitions |
| Mixing filled/outline icons | One style per hierarchy level |
| Horizontal scroll on mobile | Stack vertically |
| Text < 12px for body | Minimum 14px body text |
| Gray-on-gray text | Ensure 4.5:1 contrast |

## 14. Design Rules (PILLARS)

### PILLAR 1: User-State Adaptive Logic
Segment and adapt interfaces based on user retention stage:

| State | Behavior |
|-------|----------|
| **NEW_USER** | High whitespace, onboarding banners, curated categories only, no advanced data |
| **REPEAT_USER** | Skip onboarding, show active orders, progress tracking, contextual data |
| **SUPER_USER** | Advanced metrics, history analytics, personalized suggestions |

### PILLAR 2: Dynamic Intent Search Architecture
Search is a high-intent state — never show blank:
- Show **Recent Searches** for task resumption
- Show **Trending/Popular items** for discovery
- Show **Personalized Recommendations** from history
- Suggestions are visually subtle — lower cognitive load without obstructing power users

### PILLAR 3: Post-Purchase & Post-Action Reassurance
- **No data dumping** — replace raw text lists with visual timelines
- **Humanization** — show provider photos, names, call buttons
- **Visual Timeline** — linear progression with current state scannable in 1 second

### PILLAR 4: Structural Category & Card Consistency
- **No** un-styled text-only vertical stacks
- **No** mismatched imagery styles
- **Use** color-coded cards with solid background fills
- **Use** isolated icons from a unified family (Material Outlined)

### PILLAR 5: Task-Contextual Input Selection
- **LOW_PRECISION / ONETIME**: scroll wheels, dials, sliders
- **HIGH_PRECISION / REPETITIVE**: text fields, number pads, steppers
- **Banned**: sliders for exact numeric entry
